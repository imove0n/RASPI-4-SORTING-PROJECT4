IMPORTANT REMINDERS:
====================

1. GPIO PIN ALLOCATION:
   - GPIO4 (Pin 7): DHT22 - ALREADY IN USE
   - GPIO17 (Pin 11): Solenoid 1 - RESERVED
   - GPIO27 (Pin 13): Solenoid 2 - RESERVED
   - GPIO22 (Pin 15): Solenoid 3 - RESERVED
   - GPIO23 (Pin 16): IR sensor - RESERVED

2. DHT22 WIRING (Already connected):
   Pin 1 (3.3V) → DHT22 VCC
   Pin 7 (GPIO4) → DHT22 DATA
   Pin 6 (GND) → DHT22 GND

3. CAMERA CONNECTION:
   - Uses CSI connector (near HDMI)
   - Does NOT use GPIO pins
   - Blue strip faces away from HDMI

4. POWER CONSIDERATIONS:
   - Pi 4 needs 5V 3A (15W)
   - Solenoids need separate 12V supply
   - DO NOT power solenoids from Pi GPIO!

KEY CONCEPTS FROM OUR DISCUSSION:
==================================

1. SOLENOID vs SERVO Decision:
   - Chose SOLENOID because:
     * Faster (10-30ms vs 200-500ms)
     * Cheaper ($5 vs $15 each)
     * Simpler control (on/off vs angle)
     * More reliable for repetitive tasks

2. Camera Classification Approach:
   - Option A: Color-based (OpenCV HSV)
     * Faster, no training needed
     * 75-85% accuracy
   - Option B: CNN Machine Learning (TensorFlow Lite)
     * Slower, needs training data
     * 85-95% accuracy
     * RECOMMENDED for production

3. System Architecture:
   Camera → Classification → Timing → Solenoid → Bin
                ↓
           DHT22 Monitoring
                ↓
           Web Dashboard

4. Conveyor Design:
   - 3 columns (3 grades)
   - 6 rows (spacing)
   - 1 gate per column
   - Speed: ~3-5 tomatoes/second max
