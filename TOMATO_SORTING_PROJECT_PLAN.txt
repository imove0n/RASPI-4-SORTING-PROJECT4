SYSTEM DESIGN:

Conveyor Specifications:

- Each column has drop gate mechanism
- Gate options evaluated: Servo motor vs Solenoid

HARDWARE RECOMMENDATIONS:
=========================

1. GATE MECHANISM - SOLENOID (RECOMMENDED)
------------------------------------------
Decision: Use PUSH-PULL SOLENOIDS instead of servos

Reasoning:
✓ Fast actuation (10-30ms response time)
✓ Simple on/off control (perfect for drop gates)
✓ Cheaper ($2-5 per solenoid vs $8-15 per servo)
✓ Less power consumption
✓ More reliable for repetitive tasks
✓ Easy GPIO control (just HIGH/LOW signal)
✓ Better for high-speed sorting

Recommended Model:
- 12V Push-Pull Solenoid
- 10mm stroke length
- Force: 2-5N (sufficient for lightweight gate)
- Quantity: 3 solenoids (one per column)

Servo Comparison (NOT recommended for this):
✗ Slower (200-500ms response)
✗ More expensive
✗ Overkill for binary open/close operation
✗ More complex control code


2. CAMERA SYSTEM
----------------
Hardware Options:
- Option A: Raspberry Pi Camera Module v2 (8MP) - $25
- Option B: Raspberry Pi HQ Camera (12MP) - $50
- Recommendation: Start with v2, upgrade to HQ if needed

Camera Mounting:
- Position: Above conveyor belt
- Distance: 20-30cm from tomatoes
- Angle: 90° (directly overhead)
- Fixed mount (stable position critical)

Lighting (CRITICAL):
- LED ring light around camera lens
- Diffused white light (5000-6000K color temp)
- Eliminates shadows and reflections
- Optional: UV light for defect detection
- Cost: ~$10-15


3. SENSORS
----------
Trigger Sensor (detect when tomato enters scan zone):
- Recommended: IR Break-Beam Sensor
- Cost: ~$2 per sensor
- Quantity: 1-2 sensors
- Alternative: Ultrasonic sensor ($3-5)

Position Encoding:
- Optional: Rotary encoder on conveyor motor
- Helps calculate exact tomato position
- Cost: ~$5


4. CONTROL HARDWARE
-------------------
Relay Module:
- 3-channel relay board (5V trigger, 12V output)
- Controls 12V power to solenoids
- Cost: ~$5-8

Power Supply:
- 12V DC adapter (2A minimum)
- Powers solenoids via relay
- Cost: ~$8-12


5. GPIO PIN ALLOCATION
----------------------
Current Usage:
- GPIO4 (Pin 7): DHT22 sensor (ALREADY USED)

Proposed Addition:
- GPIO17 (Pin 11): Solenoid 1 - Grade A (Premium)
- GPIO27 (Pin 13): Solenoid 2 - Grade B (Standard)
- GPIO22 (Pin 15): Solenoid 3 - Reject/Processing
- GPIO23 (Pin 16): IR sensor input (tomato detection)
- GPIO24 (Pin 18): Optional second sensor
- GPIO25 (Pin 22): Relay board enable (optional)

Camera:
- CSI connector (dedicated camera port, no GPIO needed)

SOFTWARE ARCHITECTURE:
======================

1. CLASSIFICATION APPROACH
--------------------------

Option A: COLOR-BASED SORTING (Simpler, Faster)
------------------------------------------------
Technology: OpenCV with HSV color space analysis
Processing Time: 10-50ms per image
Training Required: None (rule-based)

Categories:
- Green (Unripe)
- Yellow (Turning)
- Red (Ripe)
- Dark Red/Brown (Overripe/Reject)
- Defect detection (brown spots, cracks)

Pros:
✓ No ML training needed
✓ Faster processing
✓ Works immediately
✓ Easy to debug
✓ Lower computational load

Cons:
✗ Less accurate (75-85%)
✗ Can't detect subtle defects
✗ Lighting-dependent
✗ Limited to color-only sorting


Option B: CNN IMAGE CLASSIFICATION (Better, Recommended)
---------------------------------------------------------
Technology: TensorFlow Lite (optimized for Raspberry Pi)
Processing Time: 50-200ms per image
Training Required: Yes (500+ images per class)

Categories:
- Grade A: Premium (perfect color, no defects, ideal size)
- Grade B: Standard (good color, minor defects, normal size)
- Grade C: Processing (off-color, defects, any size)
- Reject: Severe defects, rot, foreign objects

Pros:
✓ Higher accuracy (85-95%)
✓ Detects size, shape, color, defects
✓ Learns subtle patterns
✓ Can improve with more data

Cons:
✗ Requires training dataset
✗ Slower processing
✗ More complex setup
✗ Needs model optimization


RECOMMENDED: Option B (CNN) for production quality


2. SYSTEM WORKFLOW
------------------

Step 1: Tomato Detection
- IR sensor detects tomato entering scan zone
- Triggers camera capture
- Records timestamp

Step 2: Image Capture
- Pi Camera takes high-res photo
- Fixed position ensures consistency
- Image saved for logging/debugging

Step 3: Classification
- TensorFlow Lite model processes image
- Returns: Grade (A/B/C/Reject) + Confidence score
- Processing time: 50-200ms

Step 4: Conveyor Timing
- Calculate time until tomato reaches correct column
- Account for conveyor speed
- Example: 10 cm/sec speed, 30cm distance = 3 second delay

Step 5: Solenoid Activation
- Activate correct solenoid when tomato aligned
- Gate opens (10-30ms)
- Tomato drops into bin
- Gate closes after 200-300ms

Step 6: Logging & Monitoring
- Record: Grade, timestamp, confidence, temperature
- Update web dashboard statistics
- Store data for quality control

Step 7: Temperature Monitoring
- DHT22 continuously monitors storage area
- Alerts if temperature exceeds safe range
- Logs environmental data

CONVEYOR TIMING CALCULATIONS:
==============================

Critical Timing Parameters:
---------------------------
Distance from camera to gate: D (cm)
Conveyor speed: V (cm/sec)
Solenoid activation time: 10-30ms
Gate open duration: 200-300ms

Activation Delay Formula:
delay = (D / V) - (gate_open_time / 2)

Example:
- D = 30 cm
- V = 10 cm/sec
- delay = (30/10) - 0.15 = 2.85 seconds

Safety margin: Add 50ms for processing jitter


SAFETY FEATURES:
================

1. Emergency Stop Button
   - Hardware interrupt on GPIO
   - Immediately stops all solenoids
   - Pauses conveyor

2. Watchdog Timer
   - Resets if main loop hangs
   - Prevents stuck-open solenoids

3. Temperature Alerts
   - DHT22 monitors storage temp
   - Alerts if >30°C or <5°C
   - Prevents spoilage

4. Jam Detection
   - IR sensor timeout
   - Stops system if tomato stuck

5. Power Failure Recovery
   - Logs last state
   - Safe restart procedure
